# Mall: Veckorapport

## Format

- **Leverans:** Fredag kl 14:00 till #fia-analytics
- **Längd:** 500–800 ord
- **Ton:** Analytisk, insiktsfull, rekommenderande

## Struktur

### 1. Sammanfattning (3–4 meningar)

- Veckans övergripande resultat
- Mest signifikanta förändring
- Headline-KPI

### 2. Trafik & Synlighet

| KPI                       | Denna vecka | Förra veckan | Förändring |
| ------------------------- | ----------- | ------------ | ---------- |
| Sessioner                 |             |              |            |
| Organisk trafik           |             |              |            |
| Sidvisningar              |             |              |            |
| Genomsnittlig sessionstid |             |              |            |
| Bounce rate               |             |              |            |

### 3. Innehåll

- Publicerat innehåll denna vecka (antal, typ)
- Bäst presterande innehåll (trafik, engagemang)
- Brand Agent: godkänt vs. underkänt (ratio)

### 4. Leads & Konvertering

| KPI               | Denna vecka | Förra veckan | Förändring |
| ----------------- | ----------- | ------------ | ---------- |
| Nya leads         |             |              |            |
| MQL:er            |             |              |            |
| Konverteringsgrad |             |              |            |

### 5. Kampanjer

- Aktiva kampanjer och deras status
- Budget spenderat vs. budget totalt
- Cost per lead

### 6. Insikter & Rekommendationer

- 2–3 nyckelinsikter baserade på data
- Konkreta rekommendationer för nästa vecka
- Vad bör Orchestrator prioritera?

### 7. Metrics JSON

```json
[
  { "category": "traffic", "metric_name": "weekly_sessions", "value": X },
  { "category": "leads", "metric_name": "weekly_new_leads", "value": X },
  { "category": "content", "metric_name": "blog_posts_published", "value": X }
]
```

## Riktlinjer

- Jämför alltid med föregående vecka
- Flagga avvikelser > 20%
- Avsluta med framåtblickande rekommendationer
